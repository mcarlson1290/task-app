# Add Horizontal Scroll to Task Filters

## Problem
The filter bar is getting crowded and some filters may be cut off on smaller screens.

## Solution for Replit

### Add Horizontal Scrolling to Filter Container

```css
/* Add these styles to the filter container */
.filter-container {
  display: flex;
  gap: 10px;
  align-items: center;
  padding: 10px;
  overflow-x: auto; /* Enable horizontal scroll */
  overflow-y: hidden; /* Hide vertical scroll */
  white-space: nowrap; /* Prevent wrapping */
  -webkit-overflow-scrolling: touch; /* Smooth scrolling on iOS */
  scrollbar-width: thin; /* Firefox */
  padding-bottom: 15px; /* Space for scrollbar */
}

/* Style the scrollbar */
.filter-container::-webkit-scrollbar {
  height: 6px;
}

.filter-container::-webkit-scrollbar-track {
  background: #f1f1f1;
  border-radius: 3px;
}

.filter-container::-webkit-scrollbar-thumb {
  background: #888;
  border-radius: 3px;
}

.filter-container::-webkit-scrollbar-thumb:hover {
  background: #555;
}

/* Ensure filter items don't shrink */
.filter-container > * {
  flex-shrink: 0;
}
```

### Alternative: Draggable Scroll (Better UX)

```typescript
// Add drag-to-scroll functionality
const FilterBar = () => {
  const scrollRef = useRef<HTMLDivElement>(null);
  const [isDragging, setIsDragging] = useState(false);
  const [startX, setStartX] = useState(0);
  const [scrollLeft, setScrollLeft] = useState(0);

  const handleMouseDown = (e: React.MouseEvent) => {
    setIsDragging(true);
    setStartX(e.pageX - scrollRef.current!.offsetLeft);
    setScrollLeft(scrollRef.current!.scrollLeft);
    scrollRef.current!.style.cursor = 'grabbing';
  };

  const handleMouseMove = (e: React.MouseEvent) => {
    if (!isDragging) return;
    e.preventDefault();
    const x = e.pageX - scrollRef.current!.offsetLeft;
    const walk = (x - startX) * 2; // Scroll speed
    scrollRef.current!.scrollLeft = scrollLeft - walk;
  };

  const handleMouseUp = () => {
    setIsDragging(false);
    scrollRef.current!.style.cursor = 'grab';
  };

  return (
    <div
      ref={scrollRef}
      className="filter-container"
      onMouseDown={handleMouseDown}
      onMouseMove={handleMouseMove}
      onMouseUp={handleMouseUp}
      onMouseLeave={handleMouseUp}
      style={{
        cursor: 'grab',
        userSelect: 'none'
      }}
    >
      {/* All your filter elements */}
      <button className="all-tasks-btn">All Tasks</button>
      <select className="category-filter">...</select>
      <select className="status-filter">...</select>
      <select className="priority-filter">...</select>
      <input type="date" className="date-picker" />
      <button className="today-btn">Today</button>
      <button className="clear-filters-btn">Clear Filters</button>
      <input type="search" placeholder="Search tasks..." />
      <button className="new-task-btn">New Task</button>
      <button className="refresh-btn">Refresh Tasks</button>
    </div>
  );
};
```

### Mobile-Friendly Enhancement

```css
/* Add visual indicators for scrollable content */
.filter-wrapper {
  position: relative;
}

/* Fade effect on edges to indicate more content */
.filter-wrapper::before,
.filter-wrapper::after {
  content: '';
  position: absolute;
  top: 0;
  bottom: 0;
  width: 20px;
  pointer-events: none;
  z-index: 1;
}

.filter-wrapper::before {
  left: 0;
  background: linear-gradient(to right, white, transparent);
}

.filter-wrapper::after {
  right: 0;
  background: linear-gradient(to left, white, transparent);
}

/* On mobile, make scrollbar more visible */
@media (max-width: 768px) {
  .filter-container::-webkit-scrollbar {
    height: 8px;
  }
}
```

## Instructions for Replit

Tell Replit:

"Make the filter bar horizontally scrollable. The filter container should:
1. Allow horizontal scrolling when content overflows
2. Hide the horizontal scrollbar on desktop but show it on mobile
3. Add smooth scrolling behavior
4. Prevent filter elements from wrapping to next line
5. Optional: Add drag-to-scroll functionality for better desktop UX

Keep all filters in a single row and let users scroll/drag to see more filters."

## Expected Result
- All filters stay in one horizontal line
- Users can scroll horizontally to see hidden filters
- Smooth scrolling on both desktop and mobile
- Clean scrollbar styling
- Optional: Click and drag to scroll on desktop
