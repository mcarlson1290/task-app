## Issue 1: Fix False "Completed Late" Detection

The system is incorrectly marking tasks as late. Tasks should ONLY be marked late if they had a due date AND were completed after that due date.

### Fix the Late Detection Logic:

```javascript
const isTaskLate = (task) => {
  // Must be completed
  if (task.status !== 'completed') return false;
  
  // Must have BOTH a due date and completion time
  if (!task.dueDate || !task.completedAt) return false;
  
  // For TEST tasks or tasks without proper dates, return false
  if (task.dueDate === 'Not specified' || task.dueDate === '') return false;
  
  try {
    const dueTime = new Date(task.dueDate).getTime();
    const completedTime = new Date(task.completedAt).getTime();
    
    // Check if dates are valid
    if (isNaN(dueTime) || isNaN(completedTime)) return false;
    
    // Only late if completed AFTER due date
    return completedTime > dueTime;
  } catch (error) {
    // If any date parsing fails, task is not late
    return false;
  }
};
```

### Update Task Card to Only Show Late Badge When Actually Late:

```javascript
const TaskCard = ({ task }) => {
  // Only check if late for completed tasks with valid due dates
  const isLate = task.status === 'completed' && task.dueDate && task.dueDate !== 'Not specified' 
    ? isTaskLate(task) 
    : false;
    
  return (
    <div className={`task-card ${task.status}`}>
      <div className="task-header">
        <h3>{task.title}</h3>
        {/* Only show late indicator if actually late */}
        {isLate && (
          <span className="late-badge">⚠️ Late</span>
        )}
      </div>
      
      {/* Status badge */}
      <div className="status-section">
        {task.status === 'completed' && (
          <span className={`status-badge completed ${isLate ? 'late' : ''}`}>
            ✓ {isLate ? 'Completed Late' : 'Completed'}
          </span>
        )}
      </div>
    </div>
  );
};
```
